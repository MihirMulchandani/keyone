# Supabase update needed for Delete Account

To enable user self-delete from the app settings page, add a `DELETE` policy on `users`.

## Step-by-step

1. Open Supabase Dashboard.
2. Open your project.
3. Go to **SQL Editor**.
4. Create a new query.
5. Paste this SQL:

```sql
CREATE POLICY "users_delete" ON users FOR DELETE
  USING (id = auth.uid());
```

6. Run the query.
7. Verify in Table Editor -> `users` -> RLS Policies that `users_delete` exists.

## Why this is required

The app now has a **delete account** button in settings that runs:

```sql
DELETE FROM users WHERE id = auth.uid();
```

Without this policy, deletion will be blocked by RLS.

## What happens after deletion

- User profile row in `users` is removed.
- Related rows in `friends`, `messages`, and `rate_limits` are removed via FK cascade.
- User is signed out and local keys are cleared from IndexedDB by the app.
