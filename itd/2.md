## 1) Put correct env values in local project

Edit `.env.local`:

```env
NEXT_PUBLIC_SUPABASE_URL=https://amehpaalmnookmsbujba.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=YOUR_SUPABASE_ANON_KEY
NEXT_PUBLIC_SITE_URL=https://keyone-iota.vercel.app
```

Notes:
- `NEXT_PUBLIC_SUPABASE_ANON_KEY` = anon public key from Supabase.
- `NEXT_PUBLIC_SITE_URL` must be your real production URL.

---

## 2) Apply database schema in Supabase (very important)

Where to put schema:
- File in repo: `supabase/schema.sql`
- Run it in: Supabase Dashboard -> SQL Editor -> New Query

Steps:
1. Open Supabase dashboard.
2. Open your project.
3. Go to **SQL Editor**.
4. Open local file `supabase/schema.sql`.
5. Copy full file contents.
6. Paste into SQL Editor.
7. Click **Run**.

After running, verify:
- Tables created: `users`, `friends`, `messages`, `rate_limits`, `events`
- Enums created: `delete_mode`, `friend_status`
- RLS enabled on relevant tables
- Policies created successfully

---

## 3) Enable Realtime on messages table

1. Supabase -> Database -> Replication (or Realtime settings).
2. Ensure `messages` table is included in realtime publication.
3. Save.

Why: inbox live updates depend on realtime inserts.

---

## 4) Configure Google OAuth in Supabase

Google side:
1. Google Cloud -> APIs & Services -> Credentials.
2. Open your OAuth Web Client.
3. Add redirect URI:

```text
https://amehpaalmnookmsbujba.supabase.co/auth/v1/callback
```

Supabase side:
1. Supabase -> Authentication -> Providers -> Google.
2. Enable Google.
3. Paste:
   - Client ID
   - Client Secret
4. Save.

## 7) Deploy on Vercel

1. Open [https://vercel.com/new](https://vercel.com/new)
2. Import `MihirMulchandani/keyone`
3. In project settings, add environment variables:

```text
NEXT_PUBLIC_SUPABASE_URL=https://amehpaalmnookmsbujba.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=<your anon key>
NEXT_PUBLIC_SITE_URL=https://keyone-iota.vercel.app
```

4. Click Deploy.

Important:
- If Vercel gives you a different production URL, update:
  - `NEXT_PUBLIC_SITE_URL` in Vercel env
  - Supabase Site URL / Redirect URLs
  - Google authorized origins if needed

Then redeploy.

## 9) Post-deploy test checklist

1. Open production URL.
2. Click `continue with google`.
3. Complete onboarding:
   - username
   - key generation
   - key backup export
4. Open second account in another browser profile.
5. Add friend, accept request.
6. Send message in each deletion mode.
7. Verify inbox realtime updates.
8. Verify key import/export works.

---

## 10) Common deployment failures and fixes

- **OAuth redirect mismatch**
  - Fix Google redirect URI and Supabase URL Configuration values.

- **Login loop**
  - Ensure `NEXT_PUBLIC_SITE_URL` exactly matches production domain.

- **No messages appear**
  - Check RLS policies and realtime publication on `messages`.

- **Cannot send messages**
  - Confirm friend relationship is `accepted`.

- **Cannot decrypt**
  - Key mismatch or missing IndexedDB key; re-import backup.

---

## 11) Optional but recommended after going live

1. Rotate any temporary PAT tokens used during setup.
2. Add custom domain in Vercel.
3. Update `NEXT_PUBLIC_SITE_URL` to custom domain.
4. Update Supabase + Google OAuth URLs for the new domain.
5. Redeploy.
