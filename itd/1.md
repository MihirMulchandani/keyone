## 1) Create new Supabase project

1. Go to [https://supabase.com](https://supabase.com)
2. Create project (new organization/project name is fine)
3. Wait until project is ready

Then collect:

- `SUPABASE_PROJECT_URL`
  - Location: Project Settings -> API -> Project URL
  - Looks like: `https://xxxx.supabase.co`

- `SUPABASE_ANON_KEY`
  - Location: Project Settings -> API -> `anon public`
  - This becomes `NEXT_PUBLIC_SUPABASE_ANON_KEY`

Optional (only if we later need admin tasks):
- `SUPABASE_SERVICE_ROLE_KEY`
  - Location: Project Settings -> API -> `service_role` (keep secret)

Also send:
- `SUPABASE_PROJECT_REF` (the short project id, usually in URL)

---

## 2) Create new Google OAuth project

1. Go to [https://console.cloud.google.com](https://console.cloud.google.com)
2. Create a new project (example: `keyone-v2-new`)
3. APIs & Services -> OAuth consent screen:
   - User type: External
   - Fill app name + support email + developer email
4. APIs & Services -> Credentials -> Create Credentials -> OAuth client ID
   - Application type: Web application

Use this redirect URI:

`https://<SUPABASE_PROJECT_REF>.supabase.co/auth/v1/callback`

Then collect:

- `GOOGLE_OAUTH_CLIENT_ID`
- `GOOGLE_OAUTH_CLIENT_SECRET` (keep secret)

---

## 3) Configure Supabase Google provider

In Supabase:
1. Authentication -> Providers -> Google
2. Enable provider
3. Paste:
   - Google client id
   - Google client secret
4. Save

Send me confirmation:
- `GOOGLE_PROVIDER_ENABLED = yes`
